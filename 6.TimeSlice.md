# GPU TimeSlice — 时间分片共享

---

### 1. TimeSlice 配置

```yaml
# nvidia-time-slicing-config.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: time-slicing-rtx3080ti  # ConfigMap 名称
  namespace: gpu-operator       # GPU Operator 所在命名空间
data:
  # 配置名称，后续通过 ClusterPolicy 引用
  rtx3080ti: |-
    version: v1
    sharing:
      timeSlicing:
        renameByDefault: false              # 建议 false，使用默认资源名 nvidia.com/gpu
        failRequestsGreaterThanOne: true    # 建议 true，防止单个 Pod 请求超过 1 个 GPU 份额
        resources:
          - name: nvidia.com/gpu
            replicas: 2                     # 每张物理 GPU 虚拟化为 2 份（可调整为 4、8 等）
```

---

### 2. 创建 & 应用配置

```bash
# 创建 ConfigMap
kubectl apply -f nvidia-time-slicing-config.yaml

# 验证
kubectl get cm -n gpu-operator
```

将 TimeSlice 配置绑定到 ClusterPolicy：

```bash
kubectl patch clusterpolicies.nvidia.com/cluster-policy \
  -n gpu-operator --type merge \
  -p '{"spec": {"devicePlugin": {"config": {"name": "time-slicing-rtx3080ti", "default": "rtx3080ti"}}}}'
```

执行后 ClusterPolicy 效果：

```yaml
spec:
  devicePlugin:
    config:
      default: rtx3080ti
      name: time-slicing-rtx3080ti
    enabled: true
    image: k8s-device-plugin
    imagePullPolicy: IfNotPresent
    repository: nvcr.io/nvidia
    version: v0.19.2
```

---

### 3. 为节点打标签

> 每个 GPU 节点都需要打上对应标签，使 TimeSlice 配置生效。

```bash
kubectl label node 10-60-160-242 nvidia.com/device-plugin.config=rtx3080ti --overwrite
kubectl label node 10-60-160-243 nvidia.com/device-plugin.config=rtx3080ti --overwrite
```

---

### 4. 重启 GPU Operator 组件

```bash
kubectl rollout restart daemonset/nvidia-device-plugin-daemonset -n gpu-operator
kubectl rollout restart daemonset/gpu-feature-discovery -n gpu-operator
```

---

### 5. 验证 GPU 数量变化

```bash
kubectl describe node 10-60-160-242 | grep -A 8 "Capacity"
# Capacity:
#   cpu:                12
#   ephemeral-storage:  151263856Ki
#   hugepages-1Gi:      0
#   hugepages-2Mi:      0
#   memory:             32864884Ki
#   nvidia.com/gpu:     2        # 这里变成两张显卡了（1 物理 GPU × 2 replicas）
#   pods:               110
```

---

### 6. 回滚到最初状态

```bash
# 1. 移除节点标签
kubectl label node k8s-node01.magedu.com nvidia.com/device-plugin.config-
kubectl label node k8s-node03.magedu.com nvidia.com/device-plugin.config-

# 2. 移除 ClusterPolicy 中的配置引用
kubectl patch clusterpolicies.nvidia.com/cluster-policy \
  -n gpu-operator --type merge \
  -p '{"spec": {"devicePlugin": {"config": {"name": "", "default": ""}}}}'

# 3. 重启 DaemonSet
kubectl rollout restart daemonset/nvidia-device-plugin-daemonset -n gpu-operator
kubectl rollout restart daemonset/gpu-feature-discovery -n gpu-operator
```
